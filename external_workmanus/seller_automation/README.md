# 나만의 셀러 자동화 시스템 (Windows 11 전용)

캄파(CAMPA)를 분석하여 동작 방식을 참고하되, 파이썬 기반으로 완전히 새롭게 구축한 독자적인 셀러 자동화 시스템입니다. 
도매 3사(럭키프레시, 이컨팜, 최고집)에서 상품을 수집하여 가공한 뒤, 네이버 스마트스토어 커머스 API를 통해 자동으로 상품을 등록합니다.

## 주요 특징
1. **로컬 웹 서버**: Windows 11 PC에서 백그라운드 서버로 구동됩니다.
2. **승인 기기 보안**: 서버 PC의 IP(예: `http://192.168.0.x:8000`)로 스마트폰이나 노트북에서 접속할 수 있습니다. 단, **관리자가 승인한 기기에서만** 시스템을 사용할 수 있습니다.
3. **로컬 암호화**: 도매 사이트 비밀번호와 마켓 API 키는 모두 서버 PC의 `data/seller.db`에 암호화되어 저장됩니다. (`data/secret.key`가 마스터 키입니다)
4. **Dry-Run (모의 전송)**: 네이버에 실제로 등록하기 전, 가공된 상품명/가격/고시정보가 네이버 형식에 맞게 잘 생성되는지 미리 확인할 수 있습니다.

---

## 1. 설치 및 실행 방법

### 사전 준비
- **Python 3.10 이상**이 설치되어 있어야 합니다.
- 설치 시 **"Add Python to PATH"** 옵션을 반드시 체크하세요.

### 실행 방법
1. 압축을 푼 폴더(이 `README.md`가 있는 폴더)에서 `run_server.bat` 파일을 더블클릭하여 실행합니다.
2. 검은색 명령 프롬프트 창이 뜨면서 자동으로 필수 패키지를 설치하고 서버가 기동됩니다.
3. 서버가 켜진 상태에서, 인터넷 브라우저를 열고 **`http://localhost:8000`** 에 접속합니다.
4. **초기 관리자 비밀번호**는 `admin1234` 입니다. 로그인 후 [대시보드] 하단에서 비밀번호를 반드시 변경하세요.

---

## 2. 외부 기기(스마트폰/노트북)에서 접속하기

1. 스마트폰이나 다른 노트북의 브라우저에서 서버 PC의 내부 IP로 접속합니다. (예: `http://192.168.0.15:8000`)
2. 처음 접속하면 **"미승인 기기 대기 화면"**이 나타납니다. (화면에 식별코드가 표시됩니다)
3. 서버 PC(또는 이미 승인된 기기)에서 [대시보드]의 **접속 기기 관리** 표를 확인합니다.
4. 방금 접속한 새 기기가 '대기' 상태로 떠 있습니다. **[승인]** 버튼을 누릅니다.
5. 이제 스마트폰/노트북에서 새로고침하면 로그인 화면이 나타나며, 관리자 비밀번호로 로그인하여 시스템을 쓸 수 있습니다.

## 3. 설정 및 네이버 API 키 등록

[설정] 메뉴로 이동하여 다음을 입력합니다.

### 1) 도매 사이트 계정
- 럭키프레시, 이컨팜, 최고집의 로그인 아이디/비밀번호를 입력하고 저장합니다. (최고집은 로그인 없이 수집 가능하지만 일관성을 위해 저장해둡니다)

### 2) 네이버 API 키 등록 (중요)
- **Client ID**: 네이버 커머스 API 센터에서 발급받은 ID (예: `68bbRv08SghdAzdb8t3yNN`)
- **Client Secret**: 발급받은 Secret (비밀번호처럼 입력)
- **유동 IP 주의사항**: 
  - 네이버 커머스 API는 발급 시 등록한 **허용 IP**에서만 요청을 받습니다.
  - 가정용 인터넷은 유동 IP이므로, IP가 변경되면 네이버 커머스 API 센터에 로그인하여 **허용 IP를 현재 IP로 갱신**해야 실제 등록이 가능합니다. (Dry-run은 IP 제한과 무관하게 내부적으로 테스트됩니다)

### 3) 가공 설정
- 마진율(%), 수수료(%), 반올림 단위, 상품명 말머리(예: `[당일발송]`), 금지어 등을 설정합니다.

## 4. 상품 수집 및 네이버 등록

### 수집 (Step 1)
1. [대시보드] 중앙의 **상품 수집 실행** 영역에서 수집할 도매 사이트를 체크합니다.
2. [수집 시작]을 누르면 백그라운드에서 수집이 진행됩니다.
3. 진행률 바가 100%가 되면 완료됩니다.

### 등록 (Step 2)
1. 상단 메뉴의 **[상품 목록]**으로 이동합니다.
2. 수집된 상품들이 보입니다. 네이버에 등록할 상품의 체크박스를 선택합니다.
3. 네이버 스토어의 **카테고리 ID (leafCategoryId)**를 입력합니다. (예: `50000159`)
4. **[dry-run(모의 전송)]**에 체크가 된 상태로 **[선택 상품 등록]**을 누릅니다.
5. 실제 네이버로 전송되지 않고, 로컬에서 가격 계산/이미지 URL 치환/식품 고시 정보/원산지 등이 네이버 페이로드 구조로 잘 만들어지는지 검증합니다.
6. 문제가 없다면, dry-run 체크를 해제하고 다시 [선택 상품 등록]을 눌러 **실제 등록**을 진행합니다.

## 문제 해결 (FAQ)

- **Q. 서버 창을 실수로 껐어요.**
  - A. `run_server.bat`를 다시 실행하면 됩니다. 데이터는 `data/seller.db`에 모두 안전하게 보관되어 있습니다.
- **Q. 네이버 등록 시 HTTP 400 (GW.IP_NOT_ALLOWED) 에러가 납니다.**
  - A. 현재 PC의 공인 IP가 네이버 커머스 API 센터에 등록된 허용 IP와 다릅니다. 네이버에 접속해 IP를 갱신해주세요.
- **Q. 쿠팡/지마켓/토스는 언제 되나요?**
  - A. 1차 버전은 네이버 스마트스토어 연동까지 완성되었습니다. 쿠팡 등은 추후 동일한 구조의 플러그인 형태로 추가 업데이트될 예정입니다.
