# Naver AI 환경 대응 블로그 문안·스토리 구조 가설

- Status: `hypothesis`
- Evidence: `source_required`
- Promotion: `prohibited`
- Registry/SourcePacket registration: `not_performed`
- Intended consumer: 블로그 문안·스토리 작성 담당

이 문서는 Naver 검색 알고리즘을 설명하거나 노출 상승을 보장하지 않는다. 제공된 화면 예제에서 관찰한 **읽기 구조**만 계약으로 정리한다. Naver AI의 판독·체류시간·추천·랭킹에 관한 설명은 공식 근거와 실제 성과가 없는 가설이므로 validated 지식으로 사용하지 않는다.

## 소비 순서

작성 담당은 `BLOG_COPY_SLOT_CONTRACT.json`의 10개 슬롯을 순서대로 채운다. 사실·효능·상품 정보는 별도 근거가 있을 때만 넣고, 누락 근거를 예제값으로 메우지 않는다. 마지막 자가진단에서 하나라도 `FAIL` 또는 `UNKNOWN`이면 원고 상태는 `HELD`다.

## 텍스트-이미지 결속 규칙

각 비장식 이미지마다 다음 네 항목을 기록한다.

1. `image_id`: 원고 안에서 유일한 식별자
2. `text_intent`: 인접 문장이 전달하는 의미
3. `image_intent`: 이미지에 실제로 보이는 의미
4. `match_check`: 사람 검토 결과 `pass`, `fail`, `unknown`

오버레이 문구가 본문과 같아도 배경 이미지가 해당 의미를 보여주지 않으면 결속을 통과하지 않는다. `fail` 또는 `unknown`이면 이미지를 바꾸거나 제거한다.

## 실제 경험 작성 경계

- 허용: 관찰 시점·상황·선택 이유·확인한 사실·한계를 구체적으로 기록
- 조건부: 타인의 사례는 출처와 허가 범위를 밝히고 인용과 해석을 분리
- 금지: 사용하지 않은 상품의 사용감, 구매하지 않은 상품의 구매 과정, 관측하지 않은 성과를 1인칭 경험으로 생성
- 금지: 개인 경험을 모든 독자에게 적용되는 효능이나 사실로 일반화

## 검색어형 FAQ 작성 경계

질문은 독자가 실제로 입력할 법한 완전한 문장으로 쓰고, 답변은 `결론 → 조건 → 근거 → 한계` 순서로 구성한다. 근거가 없으면 답을 창작하지 말고 `source_required`로 반환한다.

## 최종 자가진단 체크리스트

- [ ] 첫 3~5줄에 독자 상황, 목적, 결론, 범위가 있다.
- [ ] 본문 상단 또는 중단에 근거 추적 가능한 비교·요약표가 있다.
- [ ] 경험의 주체·조건·관찰·한계가 확인된다.
- [ ] 모든 비장식 이미지의 의미가 인접 텍스트와 일치한다.
- [ ] 글이 선언된 카테고리와 실제 전문 범위에 맞는다.
- [ ] FAQ 질문이 자연어 검색 문장이고 답변에 근거와 한계가 있다.
- [ ] 광고·협찬·제공·제휴·체험·내돈내산 상태가 실제와 일치한다.
- [ ] 선크림·SPF·피부 효능 등 예제값이 기본값이나 사실 주장으로 남지 않았다.
- [ ] Naver 알고리즘 작동·노출·성과를 검증 사실로 단정하지 않았다.
- [ ] 하나의 `FAIL` 또는 `UNKNOWN`도 없다.

## 승격 금지선

이 산출물은 Pattern registry 항목이 아니다. 향후 패턴 후보화하더라도 실제 `source_claim_ids`, content QA, real performance, 명시적 human approval, 실패·rollback 기준이 모두 확인되기 전에는 `VERIFIED` 또는 `PROMOTED`로 승격할 수 없다.
