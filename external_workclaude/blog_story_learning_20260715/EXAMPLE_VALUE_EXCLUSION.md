# 예제값 제외 및 비전파 계약

## 판정

제공 이미지의 상품·피부·검색 알고리즘 관련 내용은 **구조를 보여주기 위한 예제값**이다. 사실 학습, 상품 분석, 효능 주장, 검색 랭킹 주장 또는 운영 기본값으로 저장하지 않는다.

## 학습 허용 범위

- 첫 3~5줄 요약의 위치와 역할
- 본문 상단·중단 비교표의 위치와 열 구조
- 텍스트 의도와 인접 이미지 의도의 결속 방식
- 실제 경험을 주체·상황·관찰·한계로 분리하는 방식
- 독자 상황과 목적을 명시하는 방식
- 카테고리 범위와 작성 일관성을 확인하는 방식
- 검색어형 FAQ의 질문·답변 구조
- 광고·협찬·제품 제공·제휴·체험·내돈내산 공개 슬롯
- 최종 자가진단의 fail-closed 방식

## 학습·전파 금지값

- 선크림, SPF, PA 등 자외선 차단 관련 수치와 등급
- 피부 타입, 피부 효능, 미백, 주름, 보습, 발림성, 지속시간, 세안 관련 주장
- 성분, 비건, 인증, 안전성, 모든 피부 사용 가능성 주장
- 상품명, 가격, 구매처, 배송, 구성품, 사용 후기와 개인·가족 경험 예시
- 음식·여행 등 배경 사진 자체의 사실값
- Naver AI가 체류시간을 측정하거나 복사 글을 감점한다는 주장
- Naver AI가 텍스트와 이미지를 판독·매칭한다는 작동 메커니즘 주장
- 표를 추천 스니펫·정답 화면으로 채택한다는 주장
- 특정 구조가 신뢰도·노출·랭킹·성과를 높인다는 주장

위 항목은 모두 `hypothesis/source_required`이며 이 자료만으로 검증할 수 없다.

## 금지 전파 대상

- SourcePacket의 검증 claim 또는 `source_claim_ids`
- Pattern registry의 `VERIFIED`·`PROMOTED` 항목
- 상품 데이터, 건강·피부 효능 데이터 또는 production copy 기본값
- Naver 검색 알고리즘 설명과 랭킹 보장 문구
- 자동 게시·API·WorkflowEngine 입력

## 안전한 변환 규칙

1. 원문에서 예제값과 구조 신호를 분리한다.
2. 예제값은 `[대상 A]`, `[독자 상황]`, `[근거 있는 관찰]` 같은 중립 placeholder로 치환한다.
3. 실제 원고에서 값이 필요하면 별도 출처와 권리·시점·한계를 확인한다.
4. 근거가 확인된 사실만 별도 claim으로 작성한다. 이 예제 묶음은 claim 근거로 사용하지 않는다.
5. 근거, QA, 실제 성과, 사람 승인이 없으면 패턴 승격이나 자동 적용을 중지한다.

## 명시적 제외 자산

첫 번째 묶음의 `1-Photo-1.jpg`는 CardNews SOURCE B이므로 이 학습 묶음의 파생·인용·검증 대상에서 제외했다.

## UNKNOWN

원본 URL, 작성자·게시자, 게시일, 전체 원문 문맥, 저작권·재사용 허가, 실제 광고·협찬 상태, 실제 검색·체류·전환 성과는 제공 이미지로 확인되지 않았다. UNKNOWN은 승인이나 사실로 대체하지 않는다.
