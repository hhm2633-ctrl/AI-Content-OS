# AlphaCut Research

## Summary

AlphaCut is useful as a reference for a shorts production system built around templates, layouts, timelines, and rendering.

AI-Content-OS should treat AlphaCut as a structural reference, not as a manual editing product to copy.

## Useful Elements

- Template Engine
- Layout Engine
- Highlight Engine
- Timeline Engine
- Rendering Engine
- Animation Engine

## Data Structure

```text
Project -> Template -> Scene -> Layer -> Component -> Animation -> Render
```

## Adoption Priority

- Adopt first:
  - Layout
  - Template
  - Highlight
- Roadmap later:
  - Timeline
  - Animation
  - Video Renderer

## Do Not Adopt

- Manual drag editing
- User-level direct animation setting

## AI-Content-OS Direction

Use the AlphaCut structure to improve automated card news and later shorts generation. The current MVP should stay focused on automatic layout/template/highlight decisions.
