# Decision Engine

ChatGPT Work CTO가 작업의 우선순위, 실행 경로와 승인 게이트를 판단하는 기준이다.

## 1. ROI

카드뉴스 MVP와 현재 `CURRENT_TASK.md`의 목표에 직접 기여하면 Sprint 후보로 삼는다. Shorts, Blog, Commerce, Dashboard처럼 후순위 항목은 사용자의 명시적 승인과 선행조건이 없으면 `ROADMAP.md`에 유지한다.

직접 기여는 품질, 안정성, fallback 정확성, 실행시간, 운영 효율, 실제 게시 가능성 중 하나 이상을 측정 가능하게 개선하는 것을 뜻한다.

## 2. 기본 실행 경로

| 조건 | 담당 |
|---|---|
| 일반 구현, 테스트, 문서, Git | Work/Codex 기본 경로 |
| 독립적인 2차 검토가 가치 있는 고위험 설계 | Claude 선택 가능 |
| 사용자가 특정 모델이나 별도 세션을 지정 | 해당 도구를 제한적으로 사용 |

파일 수, 새 디렉터리, 리팩토링 자체는 Claude 강제 신호가 아니다.

## 3. 위험 게이트

| 위험 신호 | 처리 |
|---|---|
| `src/workflow_engine.py` 보호 구조 변경 | 사용자 명시적 승인 필요 |
| `storage/**` 스키마 변경 | 하위 호환 추가 필드를 우선 검토 |
| 모듈·클래스·함수 이름 변경 | 최소화하고 승인 필요 |
| 삭제, 강제 Git, storage 대량 정리 | 사용자 명시적 승인 후 실행 |
| API Key, 로그인, 유료 외부 서비스 | 자격증명·비용·fallback·정책 승인 필요 |
| 실제 SNS/상품 성과 주장 | 측정 가능한 실데이터 출처가 없으면 금지 |

## 4. 선택적 Claude 사용

Claude는 다음 경우에만 선택한다.

- 동일 설계를 다른 모델이 독립적으로 검토할 가치가 큰 경우
- 사용자가 Claude의 특정 장점이나 별도 세션을 명시한 경우
- Work/Codex가 같은 차단 조건을 반복하고 새로운 관점이 필요한 경우

Claude가 참여해도 Codex MCP는 기본 요구사항이 아니며, 최종 검증과 Git은 Work/Codex가 담당한다.

## 5. 완료 판단

- 관련 도메인 스킬의 계약을 지킨다.
- 위험 기반 테스트와 compile을 통과한다.
- Sprint 끝에는 `py -m src.main`과 `workflow_completed`를 확인한다.
- 의미 있는 변경은 프로젝트 문서를 갱신한다.
- 기능 변경과 운영체계 변경을 별도 커밋으로 분리한다.
